Help

A quick reference for the risk model, ID formats, and webhook payloads.

What is Marvin?

Marvin tracks browser and editor extensions and scores them for risk. It downloads the actual extension package and inspects the code, combined with store metadata, to produce a 0–100 risk score across six signals.

Adding extensions

Paste either a full store URL or a bare extension ID on the add page. The store is auto-detected from URLs.

StoreID formatExample
chrome 32-char alphanumeric cjpalhdlnbpafiamejdnhcphjbkeiagm
vscode publisher.extensionName ms-python.python
edge GUID-like string jmjflgjpcpepeafmmgdpfkogkghcpiha

Risk score signals

Each extension is scored 0–100 across six signals. Higher = riskier.

SignalMaxSourceWhat it measures
Permissions 25packageTiered danger of declared permissions
Popularity 20store Low installs or sudden drops
Publisher 15store Ownership change, verification, generic names
Staleness 15store Time since last update
Code behaviour 15packageeval / remote code / obfuscation
External domains 10packageUnknown URLs in code
low 0–24 medium 25–49 high 50–74 critical 75–100

Users & accounts

Marvin is multi-user. Each user maintains their own isolated list of monitored extensions — adding for one user has no effect on another.

  • Admin creates users via admin / users. There is no self-registration.
  • Change your password or toggle dark mode under the username menu in the top bar.
  • Admins can delete users — all their extensions, alert rules, and webhooks are removed.

Alerts & webhooks

Configure webhook destinations and alert rules under account / alerts. Marvin POSTs a JSON payload to your webhook URL whenever a monitored extension changes.

Event typeTriggers when
risk_level_changeRisk band changes (low / medium / high / critical)
publisher_change Publisher name differs from previous fetch
permission_changeManifest permissions added or removed
new_version Version string changes
Example payload
{
  "text": "Marvin: uBlock Origin risk level changed medium → high",
  "event": "risk_level_change",
  "extension": {
    "id": 42,
    "name": "uBlock Origin",
    "store": "chrome",
    "store_url": "https://chromewebstore.google.com/detail/...",
    "marvin_url": "https://your-instance/extensions/42"
  },
  "change": { "old": "medium", "new": "high" },
  "risk_score": 55
}

marvin_url is only included when MARVIN_APP_BASE_URL is set.

Background refresh

Extensions on the watchlist are automatically re-fetched every MARVIN_FETCH_INTERVAL_MINUTES minutes (default: 60). Re-scoring detects publisher changes, install count drops, and new code patterns. Toggle watchlist per extension on the detail page.
