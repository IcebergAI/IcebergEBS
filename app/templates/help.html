what is marvin?

Marvin tracks browser and editor extensions and scores them for risk. It downloads the actual extension package and inspects the code, combined with store metadata, to produce a 0–100 risk score.

adding extensions

Paste either a full store URL or a bare extension ID on the add page. The store is auto-detected from URLs.

storeID formatexample
chrome 32-char alphanumeric cjpalhdlnbpafiamejdnhcphjbkeiagm
vscode publisher.extensionName ms-python.python
edge GUID-like string jmjflgjpcpepeafmmgdpfkogkghcpiha
risk score methodology

Each extension is scored 0–100 across six signals. Higher = riskier.

signalmaxsourcewhat it measures
permissions25package Tiered danger: critical (<all_urls>, debugger, nativeMessaging) → 25; high (cookies, history, tabs) → 15; medium → 7
popularity20store <100 installs → 16; <1k → 8; <10k → 4. Sudden drop >30% → +10
publisher15store Publisher changed between fetches → +8; unverified → +4; generic name → +3
staleness15store 3+ years old → 15; 2+ → 11; 1+ → 7; 6+ months → 4
code behaviour15package eval() / new Function() → +8; remote fetch calls → +5; obfuscation score ≥6 → +5
external domains10package 0 unknown domains → 0; 1–2 → 3; 3–5 → 6; 6+ → 10
low 0–24 medium 25–49 high 50–74 critical 75–100
users & accounts

Marvin is multi-user. Each user has their own isolated list of monitored extensions — adding an extension for one user has no effect on another user's list.

  • Admin creates users via admin / users. There is no self-registration.
  • Change your password under the username menu → change password.
  • Admins can delete users — all their extensions, alert rules, and webhook destinations are removed.
alerts & webhooks

Configure webhook destinations and alert rules under account / preferences. Marvin will POST a JSON payload to your webhook URL whenever a monitored extension changes.

event typetriggers when
risk_level_changeRisk band changes (low / medium / high / critical)
publisher_changePublisher name differs from previous fetch
permission_changeManifest permissions added or removed
new_versionVersion string changes

Rules can be scoped to all extensions or a specific extension. Each rule can be independently enabled or disabled. Disabling a destination silences all rules that use it.

Example webhook payload:

{
  "event": "risk_level_change",
  "extension": {
    "id": 42,
    "name": "uBlock Origin",
    "store": "chrome",
    "store_url": "https://chromewebstore.google.com/detail/...",
    "marvin_url": "https://your-instance/extensions/42"
  },
  "change": { "old": "medium", "new": "high" },
  "risk_score": 55
}

marvin_url is only included when MARVIN_APP_BASE_URL is set. Use the test button next to each destination to send a sample payload and verify delivery.

background refresh

Extensions on the watchlist are automatically re-fetched every MARVIN_FETCH_INTERVAL_MINUTES minutes (default: 60). Re-scoring detects publisher changes, install count drops, and new code patterns. Toggle watchlist per extension on the detail page.
